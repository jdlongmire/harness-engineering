# Evaluations

Empirical validation and benchmarks for harness engineering patterns.

This directory holds evaluation designs, results, and analysis for testing claims made by the 4M Model and the broader harness engineering programme. The programme's theoretical claims require empirical confirmation to be progressive in the Lakatosian sense (see [PROGRAMME.md](../PROGRAMME.md#5-progressive-vs-degenerating-how-to-tell)).

## Evaluation Categories

- **Conformance testing**: Does a given harness implementation satisfy the 4M architecture? (Executable tests against the conformance checklist.)
- **Comparative studies**: 4M-structured harnesses vs. unstructured alternatives on task completion, safety, maintainability, and developer productivity.
- **Module isolation**: Can each module be tested independently? Do changes to one module propagate to others?
- **Deployment portability**: Can a 4M-conformant harness be redeployed across model providers or deployment modes without rewriting the cognitive layer?
